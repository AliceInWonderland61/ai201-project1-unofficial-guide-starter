If you include Canada, UBC’s CPSC 436A is a similar OS course. You basically recreate this non-POSIX OS that has a very complex capabilities system for handling resources. It’s a huge workload and the only help you get is course resources since no OSs (other than the OS it’s based on) works like that. Each core acts like a separate instance of the kernel so multicore becomes a distributed systems problem. The time is very limited so you are fighting against tech debt from things that you previously wrote. It is a microkernel so most of the actual kernel code is already done but everything from memory allocation to process management is done in user space and has to be done.

I have no idea what that CMU course is like so it may be more difficult but UBC’s is definitely up there. I did an exchange at Berkeley so I can say for certain that their OS course (CS 162 I think?) is way way easier. I happen to specialize in systems so it was one of the easiest courses I’ve taken since everything is classic POSIX so every problem has already been solved by every other OS.
